## What is a process?

A process is a running program plus the execution context the operating system maintains for it: an address space, open resources, scheduling state, and identifiers.

## Creation and replacement

On Unix-like systems, process creation and program loading are deliberately separate operations.

```c
pid_t child = fork();
if (child == 0) {
  execlp("echo", "echo", "hello", NULL);
}
```

<Callout type="warning" title="Remember both outcomes">
  After a successful `fork()`, both parent and child continue from the next instruction. Their return values tell them which path they are on.
</Callout>

## Waiting and exit

The parent can collect the child’s termination status with `wait()` or `waitpid()`. Until that status is collected, an exited child remains as a zombie process.
