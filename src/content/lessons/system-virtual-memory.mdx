## The useful illusion

Virtual memory lets each process behave as though it owns a large, continuous address space. The operating system and hardware translate virtual addresses into physical locations.

## Typical regions

- **Text** stores executable instructions.
- **Data** stores initialized global values.
- **Heap** grows with dynamic allocation.
- **Stack** holds call frames and local state.

<Callout type="tip" title="Think in mappings">
  A virtual address is not a physical location. It is an input to a translation managed in page-sized units.
</Callout>

## Why pages matter

Paging makes protection, sharing, and demand loading practical. It also explains why access patterns influence performance: nearby addresses often share the same page and cache lines.
