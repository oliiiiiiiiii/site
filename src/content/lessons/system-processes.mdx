## Process model

A process is a running program together with its address space, open file descriptors, credentials, and execution state. On Unix-like systems, `fork()` creates a child process from the calling process.

```c
#include <stdio.h>
#include <unistd.h>

int main(void) {
    pid_t pid = fork();

    if (pid == 0) {
        puts("child");
    } else if (pid > 0) {
        puts("parent");
    } else {
        perror("fork");
    }
}
```

## Return values

- `0` in the child process
- the child PID in the parent process
- `-1` when process creation fails

<Callout title="Important" type="warning">
  The output order is not guaranteed. After `fork()`, the scheduler decides which process runs first.
</Callout>

## From fork to exec

Shells commonly use `fork()` to create a child, then `exec()` in that child to replace its program image with the requested command.
