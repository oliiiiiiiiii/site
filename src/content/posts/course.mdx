**課程大綱**

- performance analysis
- array + linked list
- stack + queue
- tree
- heap
- sorting
- string matching
- graph
- disjoint-set
- Red-Black tree
- B-tree

**上課方式**

上半學期是林軒田上課、下半學期是蔡欣穆上課，都會錄影所以可以回家看。
林軒田教得蠻好的。
蔡欣穆教得我本來覺得有點爛，尤其是紅黑數一直在那邊拿爺爺爸爸姪子來比喻每個節點，結果最後發現課本也是這樣比喻的。
下半學期蔡欣穆還辦了三個活動：
1. 大地遊戲：就是拿上課教的東西來玩大地遊戲，蠻有趣的。
2. 軟體工程：五六個人一組，每個人會分配到一個角色（Project Manager / Frontend Engineer / Backend Engineer / Quality Assurance），專案內容是要大家一起選一個資料結構把他 visualize，我覺得這個活動還不錯，但分組是隨機的這點超爛，聽到好多人都分到爛組員只好自己扛起整個 project。
3. kahoot：期末考前一週會有一個 kahoot 活動用 kahoot 幫大家複習下半學期的內容。

**評分標準**

| 佔比 | 項目 |
| - | - |
| 20% | 程式作業 |
| 20% | 手寫作業 |
| 25% | 期中考 |
| 25% | 期末考 |
| 10% | 活動 + in-class（總共 13% 然後 capped at 10%） |

A+ 線是寫死的 95 分，其他等地線也不一定照台大通用的那個，學期末教授會視大家的表現切。

**期中期末考、小考**

期中期末差很多。期中是林軒田負責的，基本上都是寫 pseudo code 的題目。期末是蔡欣穆負責的，會有很多操作題。

**期中期末報告**

無。

**作業內容及方式**

有程式作業跟手寫作業，然後下半學期每週的課結束後都會有一題很簡單的 in-class 題目，跟活動一起算分。
- 程式作業：「"miniHW + HW0" cap at 200」、「HW1」、「HW2」、「HW3」、「HW4」五個東西裡面選四個最高分的佔學期成績的 20%（最完美的配置應該就是寫所有 HW0、8 題 miniHW、HW1 - HW3）。
    - miniHW（一題 15 分，共 12 題）：每週一題，非常簡單，都是每週上課的內容。
    - HW0（一題 10 分、共 8 題）：學期初就給的 8 題蠻簡單的題目，會是從大一上的計程抄來的，如果有認真上計程就會都是寫過的題目。
